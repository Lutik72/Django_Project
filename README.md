# ☕ Сайт кофейни

Проект представляет собой многостраничный сайт для кофейни, разработанный на Django.

## ✨ Реализовано

### Главная страница
Страница-визитка с преимуществами заведения и случайными вдохновляющими цитатами.

### Меню
Динамические карточки позиций с категориями (Кофе, Десерты, Авторские напитки). Управление через админ-панель Django.

### Коворкинг-зона
- Описание пространства для работы
- **Тарифы** с ценами и описанием, загружаются из базы данных
- Возможность забронировать место

### Бронирование
- Форма с выбором тарифа
- При выборе тарифа отображается его описание и цена
- Заявки привязываются к пользователю
- После отправки заявка сохраняется в базе данных

### Пользователи
- **Регистрация** — создание аккаунта
- **Вход** — авторизация пользователей
- **Личный кабинет** — просмотр информации о пользователе
- **Редактирование профиля** — изменение имени и email
- **Смена пароля** — безопасная смена пароля
- **История бронирований** — просмотр всех заявок пользователя

### Восстановление пароля
Полноценная система восстановления пароля через email:
- Форма для ввода email зарегистрированного пользователя
- Автоматическая отправка письма со ссылкой для восстановления
- Форма для создания нового пароля
- Страница подтверждения успешного изменения пароля
- Письма отправляются в папку sent_emails, она создастся автоматически при первой попытке смены пороля "по почте" (должна быть в корне проекта, добавлена в .gitignore)

### Безопасность
- Страницы меню, коворкинга, бронирования доступны только авторизованным пользователям
- Валидация данных при регистрации и редактировании профиля

### Админ-панель
- Управление позициями меню
- Управление тарифами коворкинга
- Просмотр и управление заявками на бронирование

### 🧪 Тестирование
- Написаны тесты для URL, форм и представлений
- Покрытие ключевой функциональности

#### Как запустить тесты?
> ❗ Перед запуском тестов установите зависимости, а также примените миграции

```bash
# запуск всех тестов
python manage.py test

# запуск для конкретного приложения
python manage.py test apps.users

#Реализованные тесты в проекте
# запуск тестов только представлений приложения users
python manage.py test apps.users.tests.test_views --verbosity=2

# запуск тестов только ссылок приложения users
python manage.py test apps.users.tests.test_urls -v2

#запуск тестов только форм приложения users
python manage.py test apps.users.tests.test_forms -v2
```
> **--verbosity** - это уровень детализации вывода тестов

## Как запустить
### Вариант 1
1. Клонируйте репозиторий или скачайте его файлы
```bash
git clone https://github.com/Lutik72/Django_Project.git
```
2. Создайте виртуальное окружение и активируйте его
```bash
python -m venv venv # создать

sourse venv\Scripts\activate # активировать
```
3. Установите зависимости
```bash
pip install -r requirements.txt
```
4. Примените миграции
```bash
python manage.py migrate
```
5. Создайте суперпользователя для входа в админку
```bash
python manage.py createsuperuser
```
6. Запустите сервер
```bash
python manage.py runserver
```
### Вариант 2
1. Клонируйте репозиторий или скачайте его файлы
```bash
git clone https://github.com/Lutik72/Django_Project.git
```
2. В проекте есть файл ```run.sh```. Это текстовый файл с командами, которые выполняются последовательно, как если бы Вы вводили их вручную в терминале, если кратко - это скрипт автоматического запуска проекта.

С помощью него вы сразу:
1. Создадите виртуально окружение и запустите его
2. Установите зависимости
3. Создадите миграции
4. Запустите тесты для проверки работы
> ❗ Файл находится в корне проекта (папка coffee_area), производите запуск скрипта оттуда
```bash
sh run.sh #запуск скрипта
```
После запуска убедитесь что не было ошибок. Осталось только создать суперпользователя и запустить сервер.

### Если возникли ошибки
```bash
# Проверить миграции
python manage.py makemigrations
python manage.py migrate

# Собрать статические файлы
python manage.py collectstatic
```
